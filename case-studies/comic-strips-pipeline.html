The Problem Space

Why Creative AI Pipelines Are Hard

👁

Multiple Modalities

Text agents must orchestrate image generation without ever seeing images. Coordination happens entirely in language, across tools that return binary artifacts no agent can hold in context.

🎨

Style Coherence

Every panel, character, and cover must look like it belongs in the same comic. Six specialist agents, generating independently, have to produce visually unified output — without a shared canvas.

📊

Scale

A 5-issue saga = 30+ panels + 6 characters + 5 covers. That’s 50+ MB of images. Standard agent context can’t hold it — and trying to makes the pipeline choke on its own output.

🔄

The Quality Loop

Agents need to review their own visual output and iterate — catching misaligned style, missing characters, or broken layouts before the final assembly step commits them.

What These Constraints Forced

These four pressures forced five novel approaches that, together, unlocked a creative pipeline where 6 specialist agents produce multi-issue comic sagas from session transcripts — at ~1 KB of recipe state managing 50+ MB of images.

Novel Approach ①

Composition as Creative Superpower

A 70-line manifest composes three bundles, each bringing distinct creative intelligence. This isn’t a monolith — it’s a creative supply chain where every agent carries only its domain’s context.

amplifier-bundle-stories

Narrative craft. Arc selection, pacing, prose. The storyboard-writer doesn’t write stories; it delegates to storytelling specialists inside this bundle. Narrative expertise lives where it belongs.

amplifier-bundle-browser-tester

Visual QA. The compositor hands off to a visual-documenter to verify the final HTML renders correctly in an isolated browser environment. Testing as a composable capability.

comic-strips:behaviors

The pipeline itself. 6 agents, 3 tool modules, recipes. This behavior file is the conductor, not the orchestra — it wires specialisms together rather than containing them.

What This Unlocked

Each agent carries only its domain’s context. The storyboard-writer has narrative skills loaded; the panel-artist has image-prompt-engineering. No agent is bloated with all three concerns. This is what made 6 tight specialists possible instead of one overloaded generalist.

Novel Approach ②

The Routing Matrix as Creative Director

Each agent declares the model capabilities it needs. The matrix selects the best available model — making the entire pipeline provider-independent and gracefully degrading.

Agent Model Roles Why
style-curator creativegeneral Pure creative text — style selection and guide generation
storyboard-writer creativegeneral Narrative structure, panel breakdowns, dialogue
character-designer creativevisiongeneral Creates and reviews character reference sheets
panel-artist image-genvisioncreativegeneral Full quad — generate, review, prompt-craft, fallback
cover-artist image-genvisioncreativegeneral Same full quad — cover composition and typography
strip-compositor visioncreativegeneral Analyzes images for overlay placement — no generation needed

What This Unlocked

  • Provider independence: The same 6 agents work with OpenAI, Google, or both. Swap mid-pipeline if one hits rate limits.
  • Graceful degradation: Panel-artist requests [image-gen, vision, creative, general] — if image-gen fails, it still functions for review work.
  • Zero reconfiguration: Add a new provider to Amplifier and every comic agent automatically gains access to its models. No bundle changes.

Capability Matching, Not Provider Names

image-gen only on agents that actually generate. vision on all agents that inspect visual output. The matrix ensures each agent gets the model that matches its actual need — not just the cheapest available.

Novel Approach ③

Image Routing: Inspired by the Matrix

The Genesis Story

Watching the routing matrix match model capabilities to agent needs sparked an idea: what if we built the same pattern for image generation itself? The generate_image bridge tool has its own internal routing matrix — 12 models, 3 API surfaces, each registered with capability dimensions: reference_images, style_strengths, composition_strength, detail_ceiling, text_avoidance.

8-step selection algorithm:

Override?→ Provider→ Ref images→ Style match→ Detail level→ Task hint→ Cost sort→ Cheapest viable

The Novel Dimension: composition_strength

The system knows which models understand multi-character scenes, panel framing, and negative space. task_hint="composition" biases selection toward spatial-reasoning models. This isn’t “pick the cheapest” — it’s “pick the model that understands visual composition.”

Moderation Circuit Breaker

If OpenAI blocks a prompt, try Gemini. If all providers block it, return structured guidance telling agents to reimagine the scene — different camera angle, show aftermath instead of the moment. Content policy became a creative constraint, not a pipeline-killer.

Novel Approach ④

The 1.9 Million Token Wall

1,900,000 → 1,024
tokens per recipe step — before vs. after the comic:// protocol

The Story

In v7.3.0, {{research_data}} carried the full research JSON through every recipe step. By panel iteration 4, context hit 1.9M tokens. The pipeline was choking on its own output — not from complexity, but from carrying data instead of referencing it.

The comic:// URI Protocol

# Project-scoped (shared across issues) comic://project/characters/hero-name # Issue-scoped (per-issue assets) comic://project/issues/issue-001/panels/panel_01 # ~70 bytes per URI vs. megabytes of binary data Total recipe state: ~1 KB

3-Stage Architecture

Stage 1 — Text Only (~$0.50)

Research → style → storyboard → validate layouts. All cheap planning before committing to expensive image generation.

🛡 APPROVAL GATE — Human reviews the full plan before a single image is generated

Stage 2 — Characters (shared)

Cross-project reuse. Same character sheet works across multiple sagas if styles match.

Stage 3 — Per-Issue Art (sub-recipe)

issue-art.yaml is a reusable unit. issue-retry.yaml enables surgical recovery — retry one failed issue without regenerating the saga.

Novel Approach ⑤

Style Transfer Through Text

Agents learn visual styles by reading markdown files. Not fine-tuning. Not specialized models. A structured .md file is enough for every agent in the pipeline to produce visually coherent output — in any style, including styles that didn’t exist when the agents were written.

📄

29 Style Packs — One Pattern

Manga, watercolor, retro-americana, Ghibli, Sin City, cyberpunk noir. Each is a self-contained text file with canonical sections: Image Prompt Template, Color Palette, Panel Conventions, Character Rendering, AmpliVerse Branding. The style-curator discovers them dynamically by listing the directory. Zero hardcoded names.

🧬

The Character Rendering Section is the Key

This section defines the shared visual DNA: face proportions, eye style, line quality, rendering technique. It is quoted verbatim into every character-designer prompt as a “STYLE COHESION” directive. Every character looks like it belongs in the same comic because they all inherit from the same text-based visual genome.

✨

Custom Styles are First-Class

Any user description — “dark gritty noir with heavy shadows” — generates a complete style guide following the same template structure. Stored and reused identically to authored packs. The style system cannot distinguish “authored” from “generated.”

🔄

Cross-Project Character Reuse

Characters are indexed by style. A character from project A can appear in project B if styles match, with needs_redesign flagged for style mismatches. Style becomes the passport for cross-saga continuity.

The Infinite Extensibility Guarantee

Adding a new style = dropping a .md file. Zero agent code changes. Zero recipe changes. Zero bundle updates. The pipeline can produce in any visual style — including styles that didn’t exist when the agents were written. Style extensibility is infinite, bounded only by someone writing a markdown file.

Lessons Learned

Road Bumps → What We Hardened

Shadow-Operator’s 43-Minute Container Gauntlet

First shadow test of the comic pipeline. Container refused to build — dependencies tangled, config paths missing inside the sealed environment. Second attempt: another collapse. Third try: switched to general-purpose container, configured from scratch.

36
Container ops
43
Minutes
79
Tool calls

What was hardened: “Error out clearly rather than falling back to host environment.” The pipeline now has an explicit design rule: never modify the host. Isolation failure is loud, not silent.

The Cache Coherence Trap

The install-state.json vs cache/ mismatch bug — reset cleared the cache but not the state file, creating a “phantom installed” condition. ModuleActivator thought modules were installed but the code was gone.

What was hardened: “State tracking files must live in cache/ if they track cached data.” The bug literally wrote the co-location policy.

The Forcing Function

Shadow tests failed twice on premature fixes — forced the team to understand root causes (session creation flow bypassing AppModuleResolver) before declaring success.

What was built: The validate-storyboard step (v7.6.0) — a closed feedback loop that catches layout errors before expensive image generation begins. Fail cheap, not expensive.

The Pattern

Every road bump became a guardrail. The pipeline is robust because it was built through failure — not despite it. “Prove it in isolation first” is now the first principle of the whole system.

Takeaways

Patterns That Unlock Creative AI

1

Compose creative capabilities, don’t monolith them

Three bundles, each bringing distinct intelligence, composed through a 70-line manifest. Six tight specialists instead of one overloaded generalist. Each agent carries only its domain’s context.

2

Route by capability, not by provider name

Agents declare what they need; the system adapts. Works for both agent models (routing matrix) AND image generation (image routing matrix). Swap providers without touching agent code.

3

Reference, don’t carry

The comic:// protocol turned 1.9M tokens into ~1 KB by replacing binary blobs with 70-byte URIs. Binary data is never in context — only pointers to where it lives.

4

Teach through text, not training

Style transfer through markdown files means infinite extensibility. No fine-tuning required. The pipeline can produce in any visual style, including styles that didn’t exist when the agents were written.

5

Fail in isolation, harden in production

Shadow testing is painful but every failure became a guardrail. 43 minutes of container gauntlet prevented weeks of “works on my machine” bugs and wrote three design principles.

6

Gate the expensive work

Approval gates and validation steps before image generation save time, money, and context. ~$0.50 of text planning before committing to $5–$20 of image generation is the right order of operations.
